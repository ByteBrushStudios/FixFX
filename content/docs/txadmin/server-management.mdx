## History

The History tab maintains records of all player actions and admin interventions.

### Action Types

- **Warns** - Player warnings issued
- **Bans** - Bans applied to players
- **Kicks** - Players kicked from server
- **Other Actions** - Various administrative actions

### Search Options

- **Search by Action ID** - Find specific actions
- **Action Type Filter** - Filter by warn, ban, kick, etc.
- **Admin Filter** - See which admin took action
- **Sortable Columns**:
  - Action
  - Player
  - Reason
  - Author
  - Date/Time

This creates an accountability log for:

- Verifying fairness of bans
- Training new admins
- Documenting rule enforcement
- Appeals and disputes

## Player Drops

Analytics for unexpected player disconnections.

### Metrics

- **Expected Player Drops** - Players who left normally
- **Unexpected Player Drops** - Crashes or network issues
- **Resource Kicks** - Players removed by resource scripts
- **Period Overview** - Time-based statistics

### Use Cases

- Identify if specific resources are causing crashes
- Track network stability issues
- Spot problematic resources that need optimization

<Callout type="info">
  Frequent crashes in a specific resource? That resource likely needs
  optimization or bug fixes.
</Callout>

## Whitelist

Manage which players can join your server when whitelist is enabled.

### Whitelist Modes

- **Approved License** - Server requires whitelist (recommended)
- **Manual Mode** - Whitelist disabled (optional)

### Whitelist Management

- **Whitelist Requests** - Players requesting whitelist access
  - View pending requests
  - Approve or deny applications
  - Search by player name

- **Approved Whitelists** - Whitelisted players
  - Shows approved but not yet joined
  - Shows already joined players

### Player Identifier Types

Whitelist accepts any of these identifiers:

- `steam:` - Steam ID
- `fivem:` - FiveM ID
- `license:` - Game license
- `license2:` - License2
- `discord:` - Discord ID
- `live:` - Xbox Live ID

### Setting Up Whitelist

1. Enable whitelist mode in Settings â†’ Player Manager
2. Add players to whitelist individually or in bulk
3. Players will see a whitelist rejection on connect
4. Accept their applications or manually add their IDs

<Callout type="info">
  **Best Practice:** Use Discord integration to automate whitelist approvals
  through bot commands.
</Callout>

## Settings

Server-wide configuration accessible only to master admins.

### Available Settings

- **Server Name/Description** - Modify how server appears in browser
- **Whitelist Mode** - Enable/disable whitelist
- **Player Manager** - Configure player limits and join restrictions
- **Logging** - Configure what gets logged
- **Security** - Password policies, API keys, etc.
- **Integration** - Discord bot, webhook configuration
- **Customization** - Custom commands, server-specific settings

<Callout type="info">
  Settings changes take effect immediately for most options. Some may require
  server restart.
</Callout>

## Best Practices

### Daily Management

1. **Check Dashboard** - Monitor uptime and performance
2. **Review Players** - Monitor for problematic players
3. **Check Console** - Look for resource errors
4. **Monitor Performance** - Watch frame times

### Weekly Tasks

1. **Review History** - Audit admin actions
2. **Check Whitelist** - Process pending applications
3. **Analyze Drops** - Investigate crashes
4. **Backup Config** - Export server configuration

### Monthly Tasks

1. **Update Resources** - Check for resource updates
2. **Review Admins** - Verify admin permissions are still appropriate
3. **Audit Bans** - Review ban list for expired bans
4. **Performance Review** - Analyze server stats and trends

## Troubleshooting

### High Frame Times

- Check Console for errors
- Review which resource was last started
- Disable recently added resources
- Monitor Player Drops for patterns

### Players Can't Join

- Check whitelist status
- Verify server isn't full
- Look at console for connection errors
- Check firewall/port forwarding

### Resource Crashes

- Use Player Drops to identify problematic resource
- Check Console for resource errors
- Restart specific resource
- Consider updating or replacing the resource

### Admin Issues

- Verify admin still has permissions in Admin list
- Check if their identifier is correct
- Confirm they're using right login method
- Review their account settings

## Advanced Features

### Command Execution

Execute server commands through the console:

- `/say [message]` - Server-wide announcement
- `/kick [id] [reason]` - Remove player from server
- `/ban [identifier] [reason]` - Permanently ban player
- Custom resource commands - Check your resource documentation

### Player Notes

Add notes to player profiles for:

- Ban reasons
- Warnings issued
- Behavior notes
- Contact information

### Audit Trail

All admin actions are logged:

- Who did what
- When they did it
- What changed
- Why (if reason provided)

Use this for:

- Accountability
- Training new admins
- Investigating disputes
- Proving fair moderation
